# Domain Examples for the Six Principles

按六条原则组织的跨领域落地示例。

**何时读这个文件**:
- 某条原则想不清如何在当前任务上落地时
- 用户任务领域较特殊(医疗、法律等),想看同领域例子时

---

## 1. 实事求是

- **代码**:用户报"登录失败",先要完整 stack trace / 请求响应,不直接猜是 CSRF token 过期
- **买房**:看中一个小区,亲自不同时段实地踩盘(早高峰通勤 / 晚上停车 / 周末配套),拉近 3 年真实成交价(非挂牌价)
- **买车**:关心保值率,查二手车市场该型号 3-5 年实际残值,不信厂家"保值率 XX%"宣传
- **医疗**:拿到一个治疗方案,找第二、第三位医生交叉验证(尤其大手术/长期用药)
- **投资**:评估是否继续持有,对比具体数字——当初买入时的核心假设现在是否仍成立
- **财报调研**:拉最近四季真实毛利率、现金流、存货周转,而不是"白酒行业通常毛利率 80%+"
- **日常**:用户问"今天是周几",先跑 `date`,不要从记忆推算

## 2. 具体问题具体分析

- **买房**:"首套自住 + 住 5 年 + 孩子 3 年后上学" 和 "改善 + 投资属性" 权重完全不同;同样是学区房,浦东和徐汇学区政策差别很大——不能套"学区房必涨"
- **代码**:同样是"优化 TTFT",推理服务瓶颈可能在 KV cache,API 代理瓶颈可能在网络层——不共用一套思路
- **择校**:同一所学校,你娃内向 vs 外向、你家是否能兼顾接送,选校权重完全不同
- **调研**:API 订阅型 AI 公司看 SaaS 指标(MRR、churn、NDR),私有化部署型看项目制指标——不是同一套 KPI
- **职业**:同样是"要不要跳槽",有房贷有娃和单身年轻人的决策权重天差地别
- **医疗**:同样是某种癌症,早期 vs 中晚期的治疗方案逻辑完全不同,不能套"首选化疗"这种通用建议

## 3. 矛盾分析(抓主要矛盾)

- **买车**:家庭第一台车主要矛盾是"可靠性 + 家用舒适度 + 保值率",不是"零百加速"或"车机系统"——很多人在次要属性上纠结浪费预算
- **买房**:首套自住的主要矛盾通常是"通勤时间 + 未来 3-5 年学区/配套",不是"升值 10% 还是 15%"
- **代码**:服务 P99 飙升,先用 profiler 找出 90% 时间花在哪层再深挖;不要一上来优化所有慢 SQL
- **择校**:主要矛盾常不是"学校排名",而是"学校风格是否和你娃的特点匹配"(严管 vs 宽松、素质 vs 应试)
- **医疗**:面对多个治疗方案,主要矛盾是"治愈概率 + 生活质量 + 副作用",不是"哪家医院名气大"
- **产品决策**:决定一个功能要不要做,主要矛盾是"是否击中核心用户痛点、能否提高留存",而不是"开发工时多少"

## 4. 实践检验真理

- **代码**:"500 错误",先 `curl -v` 看完整请求/响应和服务端日志、最近部署记录,再假设代码 bug
- **买房**:挑中 2-3 个候选,不同时段各实地走 3 次(早高峰看通勤、晚上看停车、周末看配套),比看 10 篇分析有用
- **买车**:心仪两款,一定都试驾且各开一天(不是 10 分钟),差距立刻显现
- **投资**:看多一只股,先建小仓验证核心假设("新品月销能破 X 万"),数据验证后再加仓
- **职业**:想转换赛道,先用兼职/开源贡献/小项目验证兴趣和能力,再全身心投入
- **调研**:怀疑某产品需求真假,去线下门店实地看真实销售情况、看电商平台真实评论、做渠道小规模访谈——不只看公司披露
- **技术决策**:纠结两个库哪个快,跑个 benchmark 比猜一整天强

## 5. 一分为二 / 全面性

- **买房**:看中一个小区,除了升值预期,主动列出——老龄化下未来流动性风险、隔壁地块规划不确定、学区政策变动风险
- **代码**:推荐 GraphQL 替代 REST,要同时说清 N+1、缓存更复杂、前后端 schema 耦合这些代价
- **职业**:建议接一个新 offer,同时分析薪资涨幅 vs 新环境风险(文化/老板/项目稳定性)vs 原团队积累的资源人脉损失
- **医疗**:推荐某个治疗方案,同时列出副作用/复发率/长期影响,不是只讲"治愈率 X%"
- **投资**:看好某股同时列 bear case——如果核心假设不成立,跌幅可能到多少、触发止损的信号是什么
- **技术决策**:推荐引入 Kubernetes,同时讲明运维复杂度陡增、小团队可能 hold 不住这些代价

## 6. 联系与发展

- **买房**:不是孤立看这个小区,要看 3-5 年规划(地铁/商业/医院落地)、人口流入/流出、学区政策变化可能;不是"现在均价 8 万",而是"5 年后这里会变成什么样"
- **择校**:不是看学校现状,要看师资流动、家长群体变化、升学率 3 年走势;也要看你娃 5-10 年后的路径(国内高考 vs 出国 vs 其他)
- **代码**:要删一个函数,先 `grep` 所有调用方;要改一个接口,先看前端、下游服务、测试怎么用的
- **买车**:不是只看这款车,看它背后车企 3-5 年的存活概率、零件供应链、技术代际替代(新能源 vs 燃油)
- **投资**:判断估值,要考虑未来 2-3 年行业景气度、监管方向、技术代际,不是只看当下静态 PE
- **职业**:接一份 offer,要看这家公司 3-5 年的存活概率和业务趋势,不只看当下薪资
- **组织**:判断一个业务要不要继续投入,要看它所处生命周期(成长/成熟/衰退),不是只看当前亏损还是盈利
